Аудит кода: МодульОбъекта.bsl (Процедура ОбработкаПроведения документа "РеализацияТоваров")
================================================================================

1. Строка: ~29
   Серьёзность: Critical
   Описание: Обращение к свойству потенциального Неопределено. 
             Переменная СтрокаТовара = Товары.Найти(...) может принять значение Неопределено, 
             если номенклатура из результата запроса отсутствует в табличной части документа. 
             Обращение СтрокаТовара.Количество вызовет ошибку времени выполнения.
             Также в ТЧ могут быть дубли номенклатуры, и Найти() вернёт только первую строку, 
             что приведёт к некорректной проверке.

2. Строка: ~46
   Серьёзность: Critical
   Описание: Деление на ноль. Выражение СтрокаТЧ.Сумма / СтрокаТЧ.Количество не защищено 
             проверкой. Если Количество = 0, произойдёт ошибка времени выполнения.

3. Строка: ~13-35
   Серьёзность: Critical
   Описание: Race condition (условие гонки) при проверке остатков. 
             Чтение остатков выполняется без блокировки данных (отсутствует ДЛЯ ИЗМЕНЕНИЯ 
             в запросе или объект БлокировкаДанных). Между чтением остатков и записью движений 
             другой сеанс может списать товар, в результате остаток уйдёт в минус.

4. Строка: ~8, 31, 50
   Серьёзность: Warning
   Описание: Использование устаревшей процедуры Сообщить(). 
             В управляемом приложении рекомендуется использовать СообщениеПользователю, 
             иначе сообщения могут не отображаться пользователю или отображаться некорректно.

5. Строка: ~16-35
   Серьёзность: Warning
   Описание: Логическая ошибка в алгоритме проверки остатков. 
             Запрос читает остатки по ВСЕМ товарам на складе, а не только по товарам документа. 
             Проверка остатков должна выполняться по товарам табличной части, иначе 
             запрос избыточен, а логика проверки нарушена (цикл по остаткам вместо цикла по ТЧ).

6. Строка: ~38-52
   Серьёзность: Warning
   Описание: Формирование движений не защищено проверкой флага Отказ. 
             Если ранее в процедуре был установлен Отказ = Истина (например, при проверке 
             контрагента или остатков), код всё равно продолжает создавать движения. 
             Хотя платформа откатит транзакцию, это приводит к бесполезной нагрузке.

7. Строка: ~38-52
   Серьёзность: Info
   Описание: Отсутствует проверка заполненности Склада перед формированием движений. 
             Если Склад — пустая ссылка, движения будут записаны с пустым складом, 
             что делает их бессмысленными для анализа.
