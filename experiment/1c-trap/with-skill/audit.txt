# Аудит кода 1С

## План анализа
1. Прочитать файл МодульОбъекта.bsl полностью.
2. Проверить структуру процедуры ОбработкаПроведения.
3. Проанализировать валидацию входных данных (шапка, табличная часть).
4. Проверить корректность работы с запросами и блокировками данных.
5. Проверить обработку результатов запросов и возможные ошибки времени выполнения.
6. Проверить формирование движений и расчётных полей.
7. Проверить использование устаревших/не рекомендуемых конструкций.
8. Записать структурированный отчёт.

## Найденные проблемы

### 1. [Critical] Обращение к свойству Неопределено без проверки
- **Строка**: 30
- **Описание**: Переменная `СтрокаТовара` получает значение через `Товары.Найти(Результат.Номенклатура, "Номенклатура")`. Если номенклатура присутствует на складе (есть остаток), но отсутствует в табличной части документа, метод `Найти()` вернёт `Неопределено`. В следующей строке выполняется обращение к свойству `СтрокаТовара.Количество`, что приведёт к ошибке времени выполнения «Обращение к свойству объекта недопустимо для значения Неопределено».
- **Рекомендация**: Добавить проверку: `Если СтрокаТовара = Неопределено Тогда Продолжить; КонецЕсли;` или иным способом обработать ситуацию отсутствия строки в ТЧ.

### 2. [Critical] Деление на ноль при расчёте цены
- **Строка**: 46
- **Описание**: Выражение `СтрокаТЧ.Сумма / СтрокаТЧ.Количество` не защищено от нулевого количества. Если в табличной части документа присутствует строка с `Количество = 0`, произойдёт ошибка деления на ноль.
- **Рекомендация**: Добавить проверку перед делением: `Если СтрокаТЧ.Количество = 0 Тогда ЦенаЗаЕдиницу = 0; Иначе ЦенаЗаЕдиницу = СтрокаТЧ.Сумма / СтрокаТЧ.Количество; КонецЕсли;`

### 3. [Critical] Проверка остатков без блокировки данных (Race Condition)
- **Строки**: 13–35, 54–56
- **Описание**: Запрос остатков выполняется без установки управляемой блокировки (`БлокировкаДанных`). В многопользовательской среде между моментом чтения остатков и моментом записи движений другой сеанс может изменить остатки, в результате чего документ проведётся с отрицательными остатками, несмотря на предварительную проверку.
- **Рекомендация**: Перед выполнением запроса установить блокировку данных по регистру накопления «ТоварыНаСкладах» с указанием нужных измерений (Склад, Номенклатура) и режима блокировки «Исключительный».

### 4. [Critical] Ранняя установка Отказ и немедленный Возврат из цикла
- **Строки**: 32–33
- **Описание**: При обнаружении первой же ошибки остатка выполняется `Отказ = Истина; Возврат;`. В результате пользователь получит только одно сообщение, даже если проблемных товаров несколько. Это затрудняет исправление документа.
- **Рекомендация**: Накапливать ошибки в цикле (например, через `Отказ = Истина;` без немедленного возврата), а выход из процедуры выполнять после завершения цикла, если `Отказ = Истина`.

### 5. [Warning] Использование устаревшей процедуры Сообщить()
- **Строки**: 8, 31, 50
- **Описание**: Во всей процедуре используется процедура `Сообщить()`, которая является устаревшей в контексте управляемого приложения. Сообщения через `Сообщить()` не привязаны к полям формы, не локализуются стандартными средствами и отображаются некорректно в веб-клиенте.
- **Рекомендация**: Заменить на объект `СообщениеПользователю`. Для сообщений о незаполненных полях рекомендуется указывать путь к данным (`Сообщение.Поле = "Контрагент"`), для ошибок по строкам ТЧ — указывать путь к строке (`Сообщение.ПутьКДанным = "Объект.Товары[<Индекс>].Количество"`).

### 6. [Warning] Отсутствие проверки заполнения склада
- **Строка**: 23
- **Описание**: В запрос остатков передаётся параметр `Склад`, но в процедуре отсутствует проверка `ЗначениеЗаполнено(Склад)`. Если склад не указан (пустая ссылка), запрос вернёт остатки по всем складам, что может привести к ложному отсутствию ошибок или, наоборот, к ложному превышению остатков.
- **Рекомендация**: Добавить проверку заполнения склада аналогично проверке контрагента (см. строку 7).

### 7. [Warning] Отсутствие проверки заполнения табличной части Товары
- **Строка**: 38
- **Описание**: Если табличная часть `Товары` пустая, процедура не прервёт выполнение. Движения не сформируются, но документ будет считаться проведённым. Это может привести к появлению «пустых» проведённых документов без движений.
- **Рекомендация**: Перед формированием движений добавить проверку: `Если Товары.Количество() = 0 Тогда ... Отказ = Истина; Возврат; КонецЕсли;`

### 8. [Warning] Возможная некорректная проверка остатков при дублировании номенклатуры в ТЧ
- **Строка**: 29
- **Описание**: Метод `Товары.Найти()` возвращает только первую найденную строку. Если в табличной части документа одна и та же номенклатура встречается несколько раз, проверка остатков будет выполнена только по первой строке, а суммарное количество по документу не учтётся. Это может привести к списанию большего количества, чем есть на остатке.
- **Рекомендация**: Перед проверкой остатков свернуть табличную часть по номенклатуре (временная таблица в запросе или объектная обработка) либо выполнять поиск с учётом всех строк.

### 9. [Warning] Неинициализированная переменная ЦенаЗаЕдиницу при нулевом количестве
- **Строка**: 46
- **Описание**: Если количество равно нулю и ошибка деления на ноль не возникнет (например, при использовании безопасного деления или в другой версии платформы), переменная `ЦенаЗаЕдиницу` может остаться неинициализированной, что приведёт к непредсказуемому поведению в блоке `Если ЦенаЗаЕдиницу > 100000`.
- **Рекомендация**: Инициализировать переменную `ЦенаЗаЕдиницу = 0;` перед циклом или в ветке `Иначе` проверки количества.

### 10. [Info] Проверка остатков выполняется только по товарам, которые есть на складе, а не по всем товарам документа
- **Строки**: 28–35
- **Описание**: Цикл выполняется по результату запроса остатков. Если товар есть в документе, но отсутствует на складе (остаток = 0), такой товар не попадёт в выборку, и проверка не выявит ошибку. Однако при записи движений уйдёт в минус.
- **Рекомендация**: Изменить логику: сначала выбрать строки ТЧ документа (временная таблица), затем левым соединением получить остатки и сравнить с количеством в документе.

## Рекомендации по рефакторингу
1. **Структурировать проверки**: Вынести все проверки заполнения шапки и ТЧ в отдельную процедуру (например, `ПроверитьЗаполнениеДокумента`), а проверку остатков — в `ПроверитьОстаткиТоваров`.
2. **Использовать управляемые блокировки**: Обязательно устанавливать `БлокировкаДанных` перед чтением остатков для предотвращения race condition.
3. **Перейти на СообщениеПользователю**: Полностью отказаться от `Сообщить()` в пользу `СообщениеПользователю` с указанием путей к данным.
4. **Пересмотреть запрос остатков**: В запросе в качестве источника использовать табличную часть документа (временную таблицу `ВТ_Товары`), а остатки получать через `ЛЕВОЕ СОЕДИНЕНИЕ`. Это позволит проверить все товары документа, включая те, которых нет на остатке.
5. **Защитить от деления на ноль**: Всегда проверять знаменатель перед операцией деления.
6. **Обрабатывать результат Найти()**: Всегда проверять результат метода `Найти()` на `Неопределено` перед обращением к полям строки.
