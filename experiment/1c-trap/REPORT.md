# Эксперимент: Убойная ловушка на коде 1С

## Цель эксперимента

Проверить разницу в результатах работы агента **с навыками** (`plan-validator` + `tool-enforcer`) и **без навыков** на задаче аудита кода на языке 1С:Предприятие 8.3.

**Ключевое требование**: исключить влияние длины промпта. Оба агента получают одинаковую задачу, разница только в наличии skill-инструкций в контексте.

---

## Источник кода

Модуль `МодульОбъекта.bsl` — **синтетический** (создан специально для эксперимента).  
Представляет собой упрощённую процедуру проведения документа «РеализацияТоваров» с намеренно внесёнными багами разной сложности.

**Автор**: Создан агентом Kimi Code CLI для проекта `agent-gapa-skills`.  
**Дата создания**: 2026-04-26  
**Локация**: `experiment/1c-trap/{without-skill,with-skill}/МодульОбъекта.bsl`

---

## Закладки (intentional bugs)

| # | Баг | Тип | Ожидаемая сложность |
|---|-----|-----|---------------------|
| 1 | **Деление на ноль** — `СтрокаТЧ.Сумма / СтрокаТЧ.Количество` без проверки | Очевидный | Легко |
| 2 | **Обращение к `Неопределено`** — результат `Товары.Найти()` не проверяется перед `.Количество` | Тонкий | Средне |
| 3 | **Race condition** — чтение остатков без `БлокировкаДанных` или `ДЛЯ ИЗМЕНЕНИЯ` | Архитектурный | Сложно |
| 4 | **Устаревший `Сообщить()`** — вместо `СообщениеПользователю` | Стилевой | Легко |
| 5 | **Запрос по всем товарам склада** — вместо товаров документа (логическая ошибка) | Логический | Средне |

**Ложная ловушка**: `ЗначениеЗаполнено(Контрагент)` — на самом деле **правильная** конструкция. Проверяет и пустые ссылки, и `Неопределено`.

## История экспериментов

### Попытка v1 — confounding variable

- Агент «с навыком» получил **длинный промпт** с требованиями
- Агент «без навыка» получил **короткий промпт**
- Результат: 10 vs 7 проблем, но разница могла быть из-за длины промпта, а не навыка

### Попытка v2 — неявный skill

- Оба агента получили **одинаковый короткий промпт**
- Агент «с навыком» работал в директории, где лежал `.agents/skills/gapa-self-learning/SKILL.md`
- Результат: агент **не прочитал skill автоматически** (нет упоминаний в отчёте)
- Разница: 8 vs 7 — неубедительна, skill-файл не подгрузился в контекст

### Попытка v3 — интеграция skill в контекст ✅

- Оба агента получили **одинаковый короткий промпт** с задачей
- Агенту «с навыком» в **начало промпта** вставлены правила `gapa-self-learning` (имитация system-level skill)
- Агент «без навыка» — только задача

## Результаты v3 (финальные)

### Количественные метрики

| Метрика | Без навыка | С навыком |
|---------|-----------|-----------|
| **Найдено проблем** | 8 | **10** (+25%) |
| **Critical** | 3 | **4** |
| **Высокий/Серьёзный** | 3 | **4** |
| **Средний** | 2 | **2** |
| **Объём отчёта** | 102 строки | **149 строка** (+46%) |
| **Формат отчёта** | Простой список | **Структурированный** (БАГ-N с кодом) |
| **Рекомендации по рефакторингу** | ❌ Нет | ✅ Есть |

### Что нашёл агент с навыком, а без — нет

| # | Проблема | Серьёзность | Почему сложно заметить |
|---|----------|-------------|------------------------|
| 5 | **Отсутствие проверки заполнения Склада** | Серьёзный | Требует проверить все входные параметры процедуры |
| 10 | **Отсутствие обработки исключений** | Серьёзный | Требует думать о защите от сбоев на уровне архитектуры |

## Ключевые наблюдения

### Агент без навыка (v3)
- Нашёл все **очевидные и средние** ошибки
- Дал краткие рекомендации по исправлению
- Не заметил проблемы на уровне архитектуры (исключения, проверки входных данных)

### Агент с навыками (v3)
1. **Структурировал отчёт** по шаблону (БАГ-N с кодом, проблемой, рекомендацией)
2. **Проверил граничные условия** (пустой склад, исключения)
3. **Дал архитектурные рекомендации** — не просто «исправь строку», а «переделай структуру запроса»
4. **Сохранил единый формат** отчёта, который можно передать разработчику как ТЗ

## Честный вывод

**Skill эффективен при интеграции в контекст агента** (system prompt или явная загрузка). Простое наличие файла `.agents/skills/` в директории **не гарантирует** его применение — требуется поддержка на уровне платформы (как в Kimi Code CLI).

На средне-сложном коде с намеренными ловушками агент с интегрированным навыком:
- Нашёл **на 25% больше** проблем
- Выявил **архитектурные** риски, которые агент без навыка пропустил
- Сформировал **готовый к передаче** отчёт с рекомендациями по рефакторингу
