# Эксперимент: Adaptation (Самообучение)

## Цель

Проверить, работает ли механизм **adaptation** в GAPA: может ли агент записать успешный паттерн в память и применить его в будущей похожей задаче.

---

## Методология

Эксперимент состоит из трёх этапов:

### Этап 1: Запись паттерна (Session 1)

Агент решает задачу поиска файлов. Использует `find` для рекурсивного поиска — успешно.  
После решения записываем паттерн в `MEMORY.md`:
- Правило поиска файлов (`find` вместо `ls`)
- Правило подсчёта строк (`awk 'END {print NR}'` вместо `wc -l`)

### Этап 2a: Без чтения памяти

Новый агент (чистый контекст) получает похожую задачу: посчитать строки в файле.  
**Не просим** читать `MEMORY.md`.

### Этап 2b: С чтением памяти

Новый агент (чистый контекст) получает ту же задачу.  
**Просим** явно прочитать `MEMORY.md` перед выполнением.

---

## Ловушка

Файл `nofinal.txt` содержит 3 строки, но **без завершающего `\n`**:

```
first line
second line
third line (no newline at end)
```

- `wc -l` → **2** (не считает последнюю строку)
- `awk 'END {print NR}'` → **3** (корректно)

---

## Результаты

| Этап | Агент | Читал MEMORY.md | Использовал | Результат |
|------|-------|-----------------|-------------|-----------|
| 2a | Без памяти | ❌ Нет | `wc -l` | **2** ❌ |
| 2b | С памятью | ✅ Да | `awk 'END {print NR}'` | **3** ✅ |

---

## Ключевое наблюдение

**Adaptation работает, но требует явного чтения памяти.**

Агент не читает `MEMORY.md` автоматически просто потому, что файл лежит в проекте. Он должен быть либо:
1. Явно загружен в контекст (system prompt)
2. Явно прочитан агентом по инструкции
3. Подгружен платформой (как skills в Kimi Code CLI)

---

## Вывод для GAPA

| Компонент | Статус | Что нужно |
|-----------|--------|-----------|
| **Запись паттернов** | ✅ Работает | `MEMORY.md` пополняется успешными паттернами |
| **Применение паттернов** | ⚠️ Требует доработки | Нужна автоматическая подгрузка памяти в контекст |
| **Рефлексия** | ✅ Заложена | `gapa-self-learning/SKILL.md` содержит шаблон |

**Рекомендация**: Для полноценного adaptation нужен механизм, который **автоматически** вставляет релевантные записи из `MEMORY.md` в system prompt агента перед каждой задачей.
