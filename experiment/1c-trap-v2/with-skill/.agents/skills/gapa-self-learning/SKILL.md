# GAPA: Оптимизация самообучения

General Agent Performance & Adaptation — навык для итеративного улучшения через рефлексию,
экстракцию успешных паттернов и запись антипаттернов.

## Триггеры рефлексии

Запускать рефлексию по любому из условий:

1. **Завершение таска** — задача выполнена (успешно или с ошибкой)
2. **Порог tool calls** — накоплено 10+ вызовов инструментов за сессию
3. **Успех с нетривиальным решением** — починен сложный баг, написана сложная функция,
   применен неочевидный подход
4. **Ошибка** — любая ошибка, отказ инструмента, failed test, некорректный результат

## Процесс рефлексии

### Шаг 1. Анализ логов

Прочитать `references/reflection-template.md` и ответить на все вопросы шаблона,
анализируя последние действия и их результаты.

### Шаг 2. Принятие решения о записи

На основе анализа определить, что фиксировать:

| Ситуация | Действие |
|----------|----------|
| Успешное нетривиальное решение | Извлечь алгоритм → записать в skills |
| Ошибка или неудачный подход | Записать антипаттерн → в память |
| Рутинный успех без новизны | Краткая пометка, не фиксировать |
| Повтор ошибки из памяти | Обновить антипаттерн, усилить предупреждение |

### Шаг 3. Фиксация

Записать извлеченные знания в соответствующие хранилища проекта.

## Извлечение алгоритмов (Success Pattern Mining)

Если действие успешно и содержит переиспользуемую логику:

1. **Абстрагировать** — убрать специфику конкретного случая, оставить общий алгоритм
2. **Декомпозировать** — разбить на шаги: входные данные → действия → проверки → выход
3. **Формализовать условия применимости** — когда этот алгоритм работает, когда не работает
4. **Записать в SKILL.md или создать новый skill** — если паттерн кросс-доменный, оформить как отдельный skill

Формат записи алгоритма:

```markdown
### <Название паттерна>

**Контекст**: <когда применять>
**Вход**: <что нужно для старта>
**Алгоритм**:
1. <шаг 1>
2. <шаг 2>
3. <шаг 3>
**Проверка**: <как убедиться в корректности>
**Выход**: <результат>
**Ограничения**: <когда НЕ работает>
```

## Запись антипаттернов (Failure Pattern Mining)

Если действие привело к ошибке:

1. **Фиксировать контекст** — что делалось, какой инструмент, какие параметры
2. **Определить корневую причину** — почему произошла ошибка (не симптом)
3. **Сформулировать правило предотвращения** — конкретное действие или проверка для избежания
4. **Записать в MEMORY.md или антипаттерн-лог**

Формат записи антипаттерна:

```markdown
### <Дата> — <Краткое название>

**Контекст**: <что делалось>
**Ошибка**: <что произошло>
**Корневая причина**: <почему>
**Правило**: <конкретное правило предотвращения>
**Признаки повторения**: <как распознать, что ситуация повторяется>
```

## Интеграция с проектом

- **Глобальная память**: `MEMORY.md` в корне проекта — для кросс-проектных антипаттернов
- **Локальная память**: `.agents/skills/gapa-self-learning/references/anti-pattern-log.md` —
  для специфичных ошибок и паттернов этого навыка
- **Skills**: новые извлеченные алгоритмы оформлять как отдельные skill-директории
  в `.agents/skills/`

## Принципы

- **Конкретика лучше абстракций** — записывать проверяемые правила, а не общие советы
- **Актуальность** — при повторном использовании паттерна обновлять дату и контекст
- **Минимализм** — не фиксировать очевидное, фокус на нетривиальном опыте
- **Связность** — связывать новые паттерны с существующими, избегать дублирования
